# 轮播图组件升级实现文档

## 项目概述

本次升级将原有的简单轮播图组件替换为高级组合式轮播图组件，实现左侧主图展示区与右侧缩略图导航区的专业布局。

## 实现日期
2026-03-24

## 需求分析

### 用户需求
1. 更换现有轮播组件为组合式布局
2. 左侧主图区域占 60-70%
3. 右侧缩略图区域垂直排列 3-5 张
4. 当前选中项有明显高亮效果
5. 点击缩略图平滑切换主图
6. 支持左右箭头和触摸手势
7. 缩略图过多时支持滚动
8. 响应式设计（移动端上下布局）
9. 图片懒加载和错误处理
10. 可配置参数接口

### 额外需求
- 创建 AI Demo icon 图片库
- 规范化图片管理

## 实现细节

### 1. 核心组件 (advanced-carousel.js)

#### 构造函数
```javascript
constructor(container, options = {})
```
- 支持选择器字符串或 DOM 元素
- 支持配置参数覆盖
- 自动初始化

#### 配置参数
```javascript
{
    images: [],              // 图片数组
    autoPlay: true,          // 自动播放
    autoPlayInterval: 5000,  // 播放间隔
    transitionDuration: 500, // 过渡时间
    thumbnailsVisible: 4     // 可见缩略图数
}
```

#### 主要方法
- `init()` - 初始化
- `render()` - 渲染 DOM
- `bindEvents()` - 绑定事件
- `goToSlide(index)` - 跳转幻灯片
- `prev()` / `next()` - 切换
- `startAutoPlay()` / `stopAutoPlay()` - 自动播放控制
- `refresh()` - 刷新
- `destroy()` - 销毁

### 4. 交互实现

#### 点击缩略图
```javascript
item.addEventListener('click', () => {
    const index = parseInt(item.dataset.index);
    this.goToSlide(index);
});
```

#### 触摸滑动
```javascript
container.addEventListener('touchstart', (e) => {
    this.touchStartX = e.touches[0].clientX;
    this.stopAutoPlay();
});

container.addEventListener('touchend', () => {
    const diff = this.touchStartX - this.touchEndX;
    if (Math.abs(diff) > 50) {
        diff > 0 ? this.next() : this.prev();
    }
    this.startAutoPlay();
});
```

#### 自动轮播
```javascript
startAutoPlay() {
    if (!this.config.autoPlay || this.isPlaying) return;
    this.timer = setInterval(() => {
        this.next();
    }, this.config.autoPlayInterval);
    this.isPlaying = true;
}
```

### 5. 图片懒加载

```javascript
// 首张图片立即加载
<img src="${img.src}" loading="eager">

// 其他图片懒加载
<img src="${img.src}" loading="lazy" 
     onload="this.classList.add('loaded')"
     onerror="this.classList.add('error')">
```

### 6. 错误处理

```javascript
// CSS 中隐藏错误图片
.carousel-image.error {
    display: none;
}

// 显示加载状态
.image-loading {
    position: absolute;
    /* 显示"加载中..." */
}
```

## 性能优化

### 1. 图片优化
- 使用 WebP 格式（支持时）
- 适当压缩图片
- 首图优先加载

### 2. 渲染优化
- 使用 CSS transform 而非 position
- 使用 will-change 提示浏览器
- 避免强制同步布局

### 3. 内存管理
- 及时清理事件监听器
- 销毁时清理定时器
- 避免内存泄漏
