## 📱 页面入口

### 统一导航设计

**两个统计页面现在使用统一的左侧导航栏设计：**

```
┌─────────────────────────────────────────────────┐
│ 顶部标题栏                                       │
├─────────┬───────────────────────────────────────┤
│ 左侧    │ 主要内容区域                           │
│ 导航栏  │                                       │
│         │                                       │
│         │                                       │
└─────────┴───────────────────────────────────────┘
```

### 导航栏结构

**左侧导航栏包含：**

1. **统计分析分组**
   - 🏠 首页统计（链接到 `analytics-dashboard.html`）
   - 📁 项目详情页统计（链接到 `project-analytics.html`）

2. **项目详情页分组**
   - 下拉选择框：快速切换到指定项目的统计页面
   - 支持 6 个项目快速访问

### 访问方式

#### 方式 1：通过左侧导航栏
1. 打开任意统计页面（`analytics-dashboard.html` 或 `project-analytics.html`）
2. 点击左侧导航栏的"项目详情页统计"
3. 进入项目详情页统计页面

#### 方式 2：通过项目下拉框
1. 在左侧导航栏底部找到"项目详情页"分组
2. 从下拉框选择项目
3. 自动跳转到对应项目的统计页面

**支持的项目：**
- 项目 1 - B2B 电商平台
- 项目 2 - 电商 CRM 融合系统
- 项目 3 - BI 数据可视化平台
- 项目 4 - 企业级客户管理（CRM）系统
- 项目 5 - 新零售电商解决方案
- 项目 6 - 企业品牌官网（营销型）

#### 方式 3：直接访问
- 访问地址：`works/project-analytics.html`

#### 方式 4：URL 参数指定项目
- 访问地址：`works/project-analytics.html?project=project1`
- 支持的项目 ID：`project1` ~ `project6`

**位置：** 项目选择器下方（选择项目后显示）

- **总浏览次数：** 详情页被浏览的总次数

- **数据实时更新提示：** "数据实时更新"

**示例：**
```

│ B2B 电商平台                     📊 45 次浏览 │
│ project1                          🖱️ 128 次交互│
│                              数据实时更新    │
└─────────────────────────────────────────────┘
```

### 3. 模块热度排行

**位置：** 页面中部

**功能：**
- 展示详情页 8 个模块的曝光次数
- 按曝光次数从高到低排序
- 使用进度条可视化展示

**模块列表：**
1. 项目概览
2. 轮播图
3. 项目概述
4. 项目详情
5. 原型展示
6. 文档展示
7. 使用工具
8. 返回区域

**显示效果：**
```
🥇 项目概览                    25 次浏览
   ████████████████████ 100%

🥈 轮播图                      20 次浏览
   ████████████████ 80%

🥉 项目详情                    18 次浏览
   ██████████████ 72%

4️⃣ 原型展示                   15 次浏览
   ████████████ 60%
```

**数据解读：**
- 曝光次数越高 → 该模块越吸引 HR
- 排名靠后的模块 → 可能需要优化位置或内容
- 进度条长度 → 直观展示模块间的差距

---

### 4. 访问事件记录

**位置：** 模块热度排行下方

**功能：**
- 展示最近 50 条访问事件
- 包含事件的所有关键信息
- 支持时间排序（最新的在前）

**表格列：**
1. **时间：** 事件发生的时间（格式：YYYY-MM-DD HH:MM:SS）
2. **会话 ID：** 访客的唯一标识（显示前 20 位）
3. **事件类型：** 使用不同颜色的标签
   - 🔵 页面浏览
   - 🟢 模块曝光
   - 🟠 点击事件
   - 🟣 项目交互
   - 🩷 停留时长
   - 🔵 滚动深度
   - 🟡 文件下载
   - 🔴 外部链接
4. **模块/操作：** 具体的模块名称或操作类型

**示例数据：**
```
时间                会话 ID              事件类型      模块/操作
2026-03-26 10:30:15  sess_1711267200...  🟢 模块曝光   项目概览
2026-03-26 10:30:18  sess_1711267200...  🟢 模块曝光   轮播图
2026-03-26 10:30:25  sess_1711267200...  🟣 项目交互   page_view
2026-03-26 10:30:45  sess_1711267200...  🟢 模块曝光   项目详情
```

---

### 5. 停留时长分析

**位置：** 访问事件记录下方

**功能：**
- 展示 3 个停留时长指标
- 使用卡片式布局
- 直观对比数据

**指标说明：**

1. **总停留时长**
   - 定义：所有访客在该详情页的累计停留时长
   - 单位：秒（s）
   - 示例：`1250s`
   - 解读：总时长越长 → 项目越吸引 HR

2. **平均停留时长**
   - 定义：每次访问的平均停留时长
   - 计算公式：总时长 ÷ 访问次数
   - 单位：秒（s）
   - 示例：`85s`
   - 解读：平均时长越长 → 详情页内容越有吸引力

3. **最长停留时长**
   - 定义：单次访问的最长停留时长
   - 单位：秒（s）
   - 示例：`180s`
   - 解读：最长时长 → HR 最深入的一次了解

**显示效果：**
```
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│ 总停留时长      │ │ 平均停留时长    │ │ 最长停留时长    │
│ ⏳ 1250s        │ │ 🕐 85s          │ │ 🏆 180s         │
│ 累计访问时长    │ │ 每次访问平均时长│ │ 单次访问最长时长│
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 6. 数据导出

**位置：** 页面右上角

**功能：**
- 导出当前项目的完整数据
- JSON 格式
- 文件名包含项目 ID 和日期

**操作步骤：**
1. 选择要导出的项目
2. 点击右上角绿色按钮 **"导出数据"**
3. 浏览器自动下载 JSON 文件

**文件命名规则：**
```
project_{projectId}_{date}.json

示例：
- project_project1_2026-03-26.json
- project_project2_2026-03-26.json
```

**文件内容：**
```json
{
  "exportTime": "2026-03-26T10:30:00.000Z",

  "totalEvents": 128,

    {
      "eventType": "section_view",

      "pageType": "project-detail",
      "projectId": "project1",
      "projectName": "B2B 电商平台",
```

## 📊 数据分析场景

### 场景 1：评估项目详情页的内容质量

**步骤：**
1. 打开项目详情页统计页面
2. 选择要分析的项目
3. 查看"模块热度排行"
4. 分析各模块的曝光次数

**数据解读：**
```
项目 1 详情页模块曝光：
项目概览：25 次 ⭐⭐⭐⭐⭐
轮播图：20 次 ⭐⭐⭐⭐
项目详情：18 次 ⭐⭐⭐⭐
原型展示：15 次 ⭐⭐⭐
文档展示：8 次 ⭐⭐
使用工具：5 次 ⭐
```

**结论：**
- 项目概览和轮播图最吸引 HR（前 2 名）
- 文档展示和使用工具关注度低
- 建议：优化文档展示方式，或考虑精简使用工具模块

**优化方案：**
1. 将文档展示前置到项目详情之前
2. 增加文档预览功能
3. 使用工具改为图标展示，减少空间占用

---

### 场景 2：对比不同项目的吸引力

**步骤：**
1. 依次选择不同项目
2. 记录每个项目的浏览次数和交互次数
3. 对比数据

**示例数据：**
```
项目 1（B2B 电商）：45 次浏览，128 次交互 ⭐⭐⭐⭐⭐
项目 2（CRM 融合）：32 次浏览，95 次交互  ⭐⭐⭐⭐
项目 3（BI 可视化）：28 次浏览，76 次交互  ⭐⭐⭐
项目 4（CRM 系统）：22 次浏览，58 次交互  ⭐⭐
项目 5（新零售）：18 次浏览，45 次交互    ⭐⭐
项目 6（品牌官网）：12 次浏览，32 次交互  ⭐
```

**结论：**
- 项目 1（B2B 电商）最吸引 HR
- 项目 6（品牌官网）关注度最低
- 建议：优化项目 6 的展示方式，或考虑是否保留

---

### 场景 3：分析 HR 的浏览行为

**步骤：**
1. 选择项目
2. 查看"访问事件记录"
3. 分析事件类型和时间分布

**示例分析：**
```
HR A 的浏览路径：
10:30:15 - 浏览项目概览
10:30:18 - 浏览轮播图
10:30:25 - 浏览项目详情
10:30:45 - 浏览原型展示
10:31:00 - 浏览文档展示
10:31:30 - 离开

总停留时长：75 秒
浏览模块：5 个
```

**结论：**
- HR A 深度浏览了项目（5 个模块）
- 停留时长适中（75 秒）
- 对文档展示感兴趣（浏览了文档）

---

### 场景 4：识别高意向 HR

**步骤：**
1. 查看"停留时长分析"
2. 找出停留时长最长的访问
3. 结合"访问事件记录"分析行为

**高意向 HR 特征：**
- ✅ 停留时长 > 120 秒
- ✅ 浏览模块 > 5 个
- ✅ 查看了原型展示和文档展示
- ✅ 多次访问（通过会话 ID 识别）

**示例：**
```
会话 ID: sess_1711267200000_abc123
停留时长：180 秒 ⭐⭐⭐⭐⭐
浏览模块：7 个
查看模块：项目概览、轮播图、项目详情、原型展示、文档展示、使用工具、返回区域

结论：高意向 HR，建议重点关注
```

## 🔧 常见问题

### Q1：为什么选择项目后没有数据显示？

**原因：**
- 该项目还没有访问数据
- 数据被清除了
- 埋点代码未正确实施

**解决方案：**
1. 访问项目详情页（如 `works/work1.html`）
2. 浏览各个模块，产生访问数据
3. 刷新统计页面查看数据

---

### Q2：如何清空某个项目的数据？

**步骤：**
1. 打开项目详情页统计页面
2. 选择要清空的项目
3. 点击右上角红色按钮 **"清除数据"**
4. 确认操作

**注意：** 此操作会清空**所有项目**的数据，无法恢复！

---

### Q3：导出的数据文件在哪里？

**位置：**
- 浏览器默认的下载文件夹
- 通常在：`C:\Users\用户名\Downloads\`

**查找方法：**
1. 打开文件资源管理器
2. 进入"下载"文件夹
3. 按时间排序，找到最新的 JSON 文件

---

### Q4：如何查看实时数据？

**方法：**
1. 选择项目后，数据会自动加载
2. 如需刷新，点击右上角蓝色按钮 **"刷新数据"**
3. 或者重新选择项目

---

### Q5：数据会保存多久？

**当前方案：**
- 数据存储在浏览器的 localStorage
- 永久保存，除非手动清除
- 仅当前浏览器可见

**限制：**
- 不同浏览器数据不互通
- 清除浏览器缓存会删除数据
- 存储空间有限（约 5-10MB）

## 📈 最佳实践

### 1. 定期查看数据

**建议频率：**
- 每天：查看总浏览量和交互次数
- 每周：分析模块热度和停留时长
- 每月：导出数据进行深度分析

**对比维度：**
- 不同项目之间的数据对比
- 同一项目不同时间段的数据对比

### 3. 数据驱动优化

**优化流程：**
1. 查看数据，找出问题模块
2. 分析原因（位置、内容、设计等）
3. 制定优化方案
4. 实施优化
5. 对比优化前后的数据
6. 持续迭代

### 4. 多维度分析

**结合综合统计后台：**
- 查看首页数据
- 分析从首页到详情页的转化
- 识别 HR 的整体浏览路径
