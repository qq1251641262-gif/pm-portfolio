## 🔧 修复内容

### 问题描述
项目详情页统计页面的模块热度排行无数据显示。

### 根本原因
1. **analytics.js 加载顺序错误** - 在项目详情页中，analytics.js 在初始化代码之后加载，导致 `Analytics` 对象未定义
2. **模块曝光数据缺少 pageType 信息** - `trackSectionView` 函数没有从 DOM 元素中获取 `data-page-type` 和 `data-project-id` 信息
3. **数据筛选逻辑不完善** - 统计页面在筛选模块曝光数据时，无法正确匹配项目信息

### 修复方案

#### 1. 修复脚本加载顺序（work1.html）
```html
<!-- 先加载 analytics.js -->
<script src="../js/analytics.js"></script>

<!-- 再执行初始化代码 -->
<script>
    document.addEventListener('DOMContentLoaded', function() {
        if (window.Analytics) {
            Analytics.trackPageView();
            Analytics.trackProjectInteraction('project1', 'B2B 电商平台', 'page_view', {
                pageType: 'project-detail'
            });
        }
    });
</script>
```

#### 2. 增强 trackSectionView 函数（analytics.js）
```javascript
function trackSectionView(sectionId, sectionName, element = null) {
    // 从元素中获取 pageType 和 projectId 信息
    let pageType = 'homepage';
    let projectId = null;
    let projectName = null;
    
    if (element) {
        pageType = element.dataset.pageType || 'homepage';
        projectId = element.dataset.projectId || null;
        projectName = element.dataset.projectName || null;
    }
    
    reportData('section_view', {
        sectionId,
        sectionName,
        pageType,
        projectId,
        projectName
    });
}
```

#### 3. 优化数据筛选逻辑（project-analytics.html）
```javascript
function filterProjectData(data, projectId, projectName) {
    return data.filter(event => {
        const isProjectDetail = event.eventData?.pageType === 'project-detail' || 
                               event.pageType === 'project-detail';
        const isProjectMatch = event.eventData?.projectId === projectId || 
                              event.eventData?.projectName === projectName ||
                              event.projectId === projectId ||
                              event.projectName === projectName;
        
        // 对于 section_view 事件，只需要 pageType 匹配即可
        if (event.eventType === 'section_view') {
            return isProjectDetail && isProjectMatch;
        }
        
        return isProjectDetail && isProjectMatch;
    });
}
```

## 🐛 常见问题排查

### 问题 1：控制台显示"Analytics is not defined"
**原因：** analytics.js 加载失败或顺序错误

**解决方案：**
1. 检查 analytics.js 文件路径是否正确
2. 确认 analytics.js 在初始化代码之前加载
3. 检查浏览器控制台是否有 404 错误

### 问题 2：模块热度无数据显示
**原因：** 数据筛选逻辑错误或 pageType 不匹配

**解决方案：**
1. 检查 HTML 中的 data-page-type 是否为 "project-detail"
2. 检查 data-project-id 和 data-project-name 是否正确
3. 在统计页面控制台查看筛选后的数据：
   ```javascript
   console.log('筛选后的数据:', filterProjectData(getStoredData(), 'project1', 'B2B 电商平台'));
   ```

### 问题 3：模块名称显示为"Unknown"
**原因：** data-section-name 属性缺失或拼写错误

**解决方案：**
1. 检查 HTML 中的 data-section-name 属性
2. 确认属性名称拼写正确（区分大小写）
3. 确保所有模块都有 data-section-name 属性

### 问题 4：重复统计同一模块
**原因：** Intersection Observer 阈值设置过低

**解决方案：**
1. 调整 analytics.js 中的 threshold 值（当前为 0.3）
2. 增加防抖逻辑
3. 检查 trackSectionView 中的去重逻辑
